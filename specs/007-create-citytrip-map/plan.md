# Implementation Plan: Create Citytrip with Full Fields and Map-Based Location Picker

**Branch**: `007-create-citytrip-map` | **Date**: 2026-02-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-create-citytrip-map/spec.md`

---

## Summary

Extend citytrip creation to a three-step wizard (`/citytrips/create`) that captures all fields visible on the detail page: trip basics (Step 1), day plans with scheduled events (Step 2), and a review confirmation (Step 3). Step 2 includes a map picker modal per event using the Google Maps JS API and a new `location-picker.js` module, following the same `DotNetObjectReference` callback pattern established in feature 006. The existing `CreateTripCommand` is extended with `ImageUrl` and `List<DayPlanInput>`, and `MyCitytrips.razor` navigates to the new wizard route for creation (edit flow is unchanged).

## Key Implementation Notes

### 1. Wizard State Management

`CreateCitytrip.razor` owns all state across steps via a mutable model class used only in the Web layer:

```csharp
// Web-layer only — never passed to Features
public class DayPlanInputModel
{
    public int DayNumber { get; set; }
    public DateOnly Date { get; set; }
    public List<ScheduledEventInputModel> Events { get; set; } = new();
}

public class ScheduledEventInputModel
{
    public string EventType { get; set; } = "";
    public string Name { get; set; } = "";
    public TimeOnly StartTime { get; set; }
    public TimeOnly? EndTime { get; set; }
    public string? Description { get; set; }
    public PlaceInputModel? Place { get; set; }  // set by LocationPickerModal
}

public class PlaceInputModel
{
    public string Name { get; set; } = "";
    public double Latitude { get; set; }
    public double Longitude { get; set; }
}
```

When the user confirms at Step 3, `CreateCitytrip.razor` maps these to `DayPlanInput` records and dispatches `CreateTripCommand`.

### 2. DayPlan Auto-Generation (Step 1 → Step 2 transition)

```csharp
private void GenerateDayPlans()
{
    _dayPlans = Enumerable
        .Range(0, (_endDate.ToDateTime(TimeOnly.MinValue) - _startDate.ToDateTime(TimeOnly.MinValue)).Days + 1)
        .Select((_, i) => new DayPlanInputModel
        {
            DayNumber = i + 1,
            Date = _startDate.AddDays(i)
        })
        .ToList();
}
```

If `_dayPlans` already contains events and the date range changes (user navigates Back and modifies dates), a JS `confirm()` dialog prompts before regenerating.

### 3. Map Picker Modal Flow

```
User clicks "Pick on map" on EventEditorRow
  → WizardStep2 sets _pickingForEvent = eventRow
  → <LocationPickerModal> renders
  → OnAfterRenderAsync: JS.InvokeAsync<bool>("locationPicker.initPicker", elementId, dotNetRef, lat?, lng?)
  → User clicks map → JS calls dotNetRef.invokeMethodAsync('OnLocationPicked', lat, lng, name)
  → [JSInvokable] OnLocationPicked updates _pickedLat/_pickedLng/_pickedName
  → User clicks "Confirm" → OnConfirm(new PlaceInputModel {...}) fires
  → WizardStep2 sets eventRow.Place = confirmed place; closes modal
```

### 4. MyCitytrips.razor Navigation Change

```csharp
// Before (modal approach):
private void OpenCreateModal() { _editingTrip = null; _showFormModal = true; }

// After (wizard approach):
@inject NavigationManager Navigation
private void OpenCreateModal() => Navigation.NavigateTo("/citytrips/create");
```

The `TripFormModal` (edit flow) remains completely unchanged.

### 5. Image URL Validation

`CreateTripValidator` uses `Uri.TryCreate(url, UriKind.Absolute, out _)` — no new dependencies required.

### 6. Handler Extension

`CreateTripHandler` maps the command's `DayPlans` list to domain objects before passing to the repository:

```csharp
var dayPlans = request.DayPlans?
    .Select((dp, i) => new DayPlan(
        dp.DayNumber,
        dp.Date,
        timeframe: "",
        attractions: new List<Attraction>(),
        events: dp.Events
            .OrderBy(e => e.StartTime)
            .Select(e => new ScheduledEvent(
                e.EventType, e.Name, e.StartTime, e.EndTime, e.Description,
                e.Place is null ? null : new Place(e.Place.Name, e.Place.Latitude, e.Place.Longitude)))
            .ToList()))
    .ToList();
```
