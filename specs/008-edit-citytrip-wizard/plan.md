**Branch**: `008-edit-citytrip-wizard` | **Date**: 2026-02-25 | **Spec**: [spec.md](spec.md)

## Summary

Enable a trip owner to edit their citytrip through the same three-step wizard
used for creation. Clicking **Edit** on a trip card in `MyCitytrips.razor`
navigates to `/citytrips/edit/{Id}` — a new `EditCitytrip.razor` page that
pre-fills all wizard steps from `GetCitytripDetailQuery`, orchestrates
step-navigation with day-number event preservation on date changes, and on
confirmation dispatches an extended `UpdateTripCommand` that now carries
`ImageUrl` and `List<DayPlanInput>`. `WizardStep1.razor` gains an optional
`InitialModel` parameter (backward-compatible). `WizardStep3.razor` gains an
optional `ConfirmLabel` parameter (backward-compatible). No new NuGet packages
are required.

## Key Implementation Notes

### 1. WizardStep1 InitialModel Parameter

`WizardStep1` uses `private readonly WizardStep1Model _model = new()` — the field
is `readonly` (can't be reassigned) but the class instance is mutable. Pre-fill
copies property values in `OnInitialized`:

```csharp
[Parameter] public WizardStep1Model? InitialModel { get; set; }

protected override void OnInitialized()
{
    if (InitialModel is null) return;
    _model.Title           = InitialModel.Title;
    _model.Destination     = InitialModel.Destination;
    _model.StartDate       = InitialModel.StartDate;
    _model.EndDate         = InitialModel.EndDate;
    _model.Description     = InitialModel.Description;
    _model.MaxParticipants = InitialModel.MaxParticipants;
    _model.ImageUrl        = InitialModel.ImageUrl;
}
```

`OnInitialized` is used (not `OnParametersSet`) because the page is never
re-parameterized mid-session — using `OnParametersSet` would risk overwriting
user edits if the parent triggers a re-render.

### 2. Day-Number Event Preservation

`EditCitytrip.razor` uses a `PreserveByDayNumber` helper (replaces the simple
`GenerateDayPlans` from `CreateCitytrip.razor`):

```csharp
private static List<DayPlanInputModel> PreserveByDayNumber(
    List<DayPlanInputModel> oldPlans, DateOnly newStart, DateOnly newEnd)
{
    var days = (newEnd.ToDateTime(TimeOnly.MinValue)
              - newStart.ToDateTime(TimeOnly.MinValue)).Days + 1;
    return Enumerable.Range(0, days)
        .Select(i => new DayPlanInputModel
        {
            DayNumber = i + 1,
            Date      = newStart.AddDays(i),
            Events    = (i < oldPlans.Count)
                          ? oldPlans[i].Events
                          : new List<ScheduledEventInputModel>()
        })
        .ToList();
}
```

On initial page load, `oldPlans` is empty → generates empty day slots (same as
`CreateCitytrip.GenerateDayPlans`). On date change with existing events → copies
events by day number.

var updated = existing with

### 6. CreatorId in CitytripDetailResponse

`CitytripDetailResponse` currently does not include `CreatorId`. Add it:

```csharp
public record CitytripDetailResponse(
    int Id,
    string Title,
    string Destination,
    string ImageUrl,
    DateOnly StartDate,
    DateOnly EndDate,
    string? Description,
    int? MaxParticipants,
    string CreatorId,          // ADD
    List<DayPlanDetail>? DayPlans);
```

The handler maps `trip.CreatorId` to this field. `CitytripDetail.razor` ignores
the new field (it's not displayed there). This is the simplest fix — no extra
query needed for the ownership check in `EditCitytrip.razor`.
